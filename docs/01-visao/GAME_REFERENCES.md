# Referências de Design do Projeto

> Este documento define quais jogos servirão de inspiração para o desenvolvimento deste projeto.

## IMPORTANTE

Este projeto NÃO deve copiar:

- personagens
- inimigos
- sprites
- mapas
- músicas
- efeitos sonoros
- história
- diálogos
- nomes
- artes
- animações
- código

As referências abaixo servem APENAS para inspirar mecânicas, qualidade técnica, experiência do usuário e arquitetura dos sistemas.

O objetivo é criar uma IP (Intellectual Property) totalmente original.

---

# Objetivo Geral

Criar um jogo de plataforma 2D no estilo Metroidvania moderno, com:

- exploração livre
- mapa interconectado
- combate fluido
- movimentação extremamente responsiva
- progressão baseada em habilidades
- chefes memoráveis
- alta qualidade técnica
- desempenho multiplataforma

# Aspectos Técnicos Inspirados

## Movimentação

Inspirar-se em:

- Silksong
- Celeste
- Ori

Objetivos:

- controles extremamente responsivos
- baixa latência
- animações suaves
- aceleração natural

---

## Combate

Inspirar-se em:

- Silksong
- Dead Cells
- Nine Sols

Objetivos:

- combate rápido
- feedback visual
- feedback sonoro
- partículas
- hit stop
- hit flash

---

## Exploração

Inspirar-se em:

- Hollow Knight
- Super Metroid
- Symphony of the Night

Objetivos:

- mapa conectado
- caminhos alternativos
- áreas secretas
- backtracking inteligente

---

## Interface

Inspirar-se em:

- Hollow Knight
- Prince of Persia

Objetivos:

- HUD minimalista
- menus simples
- navegação intuitiva
- interface responsiva

---

## Qualidade Esperada

O projeto deverá priorizar:

- código limpo
- arquitetura escalável
- alta performance
- alta taxa de FPS
- baixo consumo de memória
- multiplataforma
- testes automatizados
- documentação completa

---

# Filosofia de Desenvolvimento

Todo sistema implementado deverá responder à seguinte pergunta:

"Esta implementação melhora a experiência do jogador?"

Caso a resposta seja NÃO, a implementação deverá ser revisada.

---

# Princípios

✔ Jogabilidade acima de efeitos visuais.

✔ Fluidez acima de quantidade de recursos.

✔ Performance acima de complexidade.

✔ Código limpo acima de soluções rápidas.

✔ Arquitetura escalável acima de atalhos.

✔ Manutenção simples acima de otimizações prematuras.

✔ Experiência do jogador sempre em primeiro lugar.

---

Os agentes devem usar estes jogos como referência apenas para compreender:

# Adendo do projeto (complemento à regra final)

> Adicionado durante a Fase 0 de planejamento.

- **Referência extra sugerida — co-op:** *It Takes Two* e *Salt and Sanctuary
  (co-op local)* para dinâmica cooperativa; *Core Keeper* e *Terraria* para
  netcode acessível em jogos 2D com amigos. Apenas mecânica — nunca conteúdo.
- Qualquer asset temporário de terceiros (placeholder) usado em protótipos
  internos **jamais** pode aparecer em builds públicas, trailers ou capturas.
- Nomes de criaturas, regiões e personagens devem passar por checagem de
  colisão com IPs existentes antes de serem considerados 🟢 aprovados.
